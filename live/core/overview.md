## What it is

The Core Service is a Go modular monolith that owns all business logic: groups, friends, expenses,
settlement, ledger, and todos. It exposes gRPC APIs (via connectRPC) and talks to the Auth Service
for token validation and permissions.

Repository: `github.com/fair-n-square-co/core`

## Module layout convention

Each domain module (`ledger`, `groups`, `friends`, `expenses`, `settlement`, `todos`) follows:

```
internal/<module>/
  api/          тЖР gRPC handlers + request validation; implements the proto service interface
  service/      тЖР domain logic; defines the module's public interface (the composability seam)
  repository/   тЖР DB queries; receives *pgxpool.Pool from internal/core/db
```

**Dependency direction:** `api` тЖТ `service` тЖТ `repository`. No cross-module package imports тАФ
modules talk to each other through the service interface, not by reaching into each other's packages.

## Proto contracts

Proto definitions live in the `apis` repo (`github.com/fair-n-square-co/apis`). Each module's
`api/` package imports the generated Go stubs and satisfies the gRPC server interface.

## DB layer

DB is a service-level concern, not per-module:

- Connection pool initialised in `internal/core/db/` and injected into each module's `repository/`
- Migrations in `db/core/migrations/` (goose)
- Queries in `db/core/queries/` (sqlc); generated code in `internal/core/db/sqlc/`

## Build & run

Requires: `go 1.26`, `just` (`brew install just`), `sqlc`, `goose`

```
just build   # go build ./...
just test    # go test ./...
just lint    # golangci-lint run
just run     # build + run cmd/core
```
